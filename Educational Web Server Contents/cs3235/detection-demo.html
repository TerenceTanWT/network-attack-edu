Detection (Demonstration)

In the digital age, internet users are trained to recognize and trust the green padlock symbol next the browser’s URL bar, indicating an HTTPS connection and thus the use of SSL/TLS. However, this is not necessarily the be-all-end-all when it comes to determining network security.

While the presence of SSL certificates is supposed to indicate trust, our computers can be tricked to trust certificates signed by unreputable certificate authorities. This can either happen due to our own carelessness, or, malware can do this itself. Malicious actors scan exploit this trust to instigate man-in-the-middle attacks against HTTPS without the user being any the wiser. Therefore, we should take the green padlock symbol with a grain of salt.

In order to add an extra layer of safety with regards to SSL certificates, we have created a Firefox extension that will automatically double-check the certificates of websites that you travel to. It does so by retrieving the certificate received on the browser and double-checking this against the certificate for the same website retrieved from a server hosted in the cloud. This ensures that you are correctly receiving the expected certificate for a given website.

As previously mentioned, this extension can help with discovering man-in-the-middle attacks against the HTTPS protocol. In particular, this helps detect SSL Hijacking. Imagine that Alice is trying to communicate to Bob via HTTPs. However, Alice’s communications are going through attacker Harry (e.g. via ARP poisoning). Then, Harry can pretend to be Bob when communicating with Alice by using his own SSL certificate. Meanwhile, Harry communicates with Bob normally. If Alice’s computer is not set up to challenge certificates correctly, or if Alice has trusted unsafe certificate authorities, she may overlook the fact that Harry’’s certificate is not actually correct. Note that if Harry actually has access to Bob’s private key, then there is nothing that can be done since he can then perfectly use Bob’s SSL certificate.

One real world example of this attack is the Avast Antivirus software. Avast gets the computer to trust its own certificate authority, then strips TLS/SSL traffic, performs analysis, and signs it with its own certificates. If this was an attacker, it’s clear to see why this might be troublesome - the attacker can access all communications. The browser still sees everything as HTTPS, since there is a valid, trusted SSL certificate. This would be detected by our Firefox extension. Another real world example of SSL Hijacking that was in the news recently is the Superfish software, which introduced vulnerabilities to Lenovo laptops that had this installed by default.






Installation Instructions

1. Download Firefox Nightly (only Nightly, Dev Edition, or ESR allows installation of unsigned extensions).
2. Go to about:config and set xpinstall.signatures.required to false, in order to allow installation of unsigned extensions.
3. Go to about:addons, click on the setting icon (top right), select Install Add-on From File..., and select the SSL_Hijack_Detector.zip file.
