HTTP and Invalid Certificates

Certificates are proof of identify. When you see the SSL padlock in the browser's address bar or at the bottom of the browser it means that a secure connection between your browser and the website has been established.

What happens if the SSL certificate is invalid?

An invalid SSL certificate can mean several things.

  1. Certificate has expired
  2. Certificate was authorised by an untrusted root Certificate Authority (CA)
  3. Certificate was forged

Invalid certificates cause the affected site(s) to be vulnerable to (to name a few):

  • Man-in-the-Midddle (MitM) Attacks
  • Advanced Persistent Malware
  • Attacker Encrypted Communications

This warning appears when the website is using an invalid SSL certificate.

To counter this, the team has developed a Firefox extension that alerts the user in the case an invalid SSL certificate is identified, and also if the website uses HTTP instead of HTTPS.
