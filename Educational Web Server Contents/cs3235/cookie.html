Session Cookie Stealing

Session hijacking (also known as cookie hijacking) is the exploitation of a valid computer session — sometimes called a session key — to gain unauthorized access to information or services in a computer system. In particular, it is used to refer to the theft of a magic cookie used to authenticate a user to a remote server. It has particular relevance to web developers, as the HTTP cookies used to maintain a session on many web sites can be easily stolen by an attacker using an intermediary computer or with access to the saved cookies on the victim's computer.

How does it work?

HTTP connections make use of every user's TCP connection, and it does so using what's called a cookie. The cookie usually comprises of a string containing variables that represent the state of the user's session. If an attacker were to gain access to either manipulate the cookie's data or steal a copy of this cookie, they would be able to compromise or recreate the user's session.


After successfully stealing appropriate session cookies an adversary might use the Pass the Cookie technique to perform session hijacking.

What does this mean?

Let's say you're logged on to your Facebook profile. Having selected the 'keep me logged in' checkbox when logging in, your session cookie would not be set to expire until the profile is manually logged out of. As a result, should an adversary somehow gain access to the details of the cookie, they would be able to directly access your session on a whole other device without autorisation.
